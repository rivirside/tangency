Digital Security Checkup

Run a 45-minute ritual that hardens devices, accounts, and data access for your entire team.

Security only works when it is a habit. By setting up a recurring checkup, you normalize password rotation, access audits, and incident prep. This kit makes that ritual easy to run, even if you do not have an in-house tech lead.

Focus: Security + Care Format: Checklist Time: 45 minutes Team: Everyone

Perfect for Quarterly staff meetings, volunteer summits, or pre-action briefings.

Deliverable Updated password manager, revoked access list, incident contact card.

Materials Password manager, MFA instructions, shared access log.

Outputs Clean list of tools + who can see what.

Prep checklist

  • Ensure everyone has access to a password manager (Bitwarden, 1Password, etc.).
  • Print or share the Tangency Security Worksheet with the steps outlined below.
  • Identify an on-call tech buddy who can help troubleshoot during the session.

Checkup flow

  1. Device updates (10 min). Prompt everyone to update OS + browsers and restart devices.
  2. Password rotation (15 min). Rotate credentials for priority accounts and store them in the manager. Enable multi-factor authentication everywhere.
  3. Access audit (10 min). Review docs, Slack workspaces, Airtable bases, etc. Remove anyone who no longer needs access.
  4. Incident card (5 min). Add legal, comms, and tech contacts to a printed or offline doc.
  5. Care close (5 min). End with hydration, stretch, or breathing exercise to reinforce that safety equals care.

Tips

  • Make it communal—play music, bring snacks, and normalize asking questions.
  • Schedule the next checkup before you dismiss the group.
  • Log any suspicious findings and follow up with a deeper investigation if needed.

How to use this kit

  1. Put quarterly checkups on the calendar. Invite everyone, not just tech leads.
  2. Distribute worksheets before you start. Everyone should update devices and rotate passwords simultaneously.
  3. Assign a tech buddy for the session. Have someone on-call to troubleshoot MFA or access issues.
  4. Log outcomes and next steps. Capture lingering issues and schedule deeper dives where needed.

Key terms

  • MFA (multi-factor authentication): A second verification step (app code, text, hardware key) that protects accounts even if a password leaks.
  • Password manager: A secure vault (Bitwarden, 1Password) that stores and generates unique passwords.
  • Access audit: A review of who still has access to shared drives, Slack, Airtable, etc.
  • Incident contacts: Pre-vetted legal, comms, and tech allies you call if there’s a breach or confiscation.
